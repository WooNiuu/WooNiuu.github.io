NONMEM DATASETS

Posted on 2020-05-27

进行群体药动学（PPK）分析的数据集要符合NONMEM格式。数据集的复杂程度取决于要建立的模型的复杂度，其编辑构建须在建模工作之前完成。药代动力学（PK）数据进集通常需要详细说明研究中剂量（给药事件），自变量（例如时间，体重，年龄）及因变量（即药物浓度）。药效动力学（PD）数据集可包含或不包含给药事件，数据集包含疾病进展的生物学过程或安慰剂作用等。

1. 数据集的整理

NONMEM（NM-TRAN）系统对分析数据集具有一定的格式要求，数据必须以文本（即ASCII)格式显示。数据以行和列的形式二维排列，除某些日期和时间格式可能包括字母字符外，所有NONMEM数据都为数值格式。

2. 数据集中的常见变量

2.1 TIME 时间

TIME变量值必须为非负，不可缺失的。除非有一条命令指示重置事件，允许重新开始时间。否则单个个体内的时间记录必须以递增（或相等）的TIME值进行排列。
TIME值可以是时钟时间（例如 13：30）或十进制时间格式（例如 13.5），但单个数据集中的TIME格式应该统一类型。

2.2 DATE 日期

DATE变量值不是必需的，它与TIME结合使用，构成事件记录的时间序列。DATA值可以表示为不同格式，如下表所示：

Variable name Format
DATE MM/DD/YY or MM-DD-YY
DAT1 DD/MM/YY or DD-MM-YY
DAT2 YY/MM/DD or YY-MM-YY
DAT3 YY/DD/MM or YY-DD-MM

用“ / ”或“ - ”作为分界符。使用DATE变量时，TIME值被限制在0到24之间。如果未使用DATE变量量，TIME即为自第一个时间以来的时间，跨越天的时间即TIME值大于24。

注意：控制文件中$INPUT必须指定 DATE = DROP

除了TIME和DATE定义时间顺序，数据集中包含 TSFD（time since first dose），TAD（time since the most recent dose or time after dose）在后续的图形绘制，生成拟合优度图等数据处理操作中会有用。

2.3 ID 受试者编号

ID identifier ，即受试者的标识符。数据集必须要有ID变量，且单个受试者的记录必须是连续的，如果被另一个ID号分隔，即被视为另外一个受试者。

要为数据集中的受试者标识唯一的ID值，避免混淆。

2.4 DV 因变量

DV dependent variable 值表示数据集中的观测值，PK数据集中的观测值即药物浓度。DV值在给药事件时必须缺失。

2.5 MDV

MDV missing DV，即DV是否缺失。MDV是逻辑变量，缺失DV时（给药事件），MDV=1（即true）；DV有观测值时，MDV=0（即false)。如果DV与MDV对应错误，在运行程序时会报错。

MDV不是数据集的必须项，但包含该变量会是个好做法，可以判断预期行为。
